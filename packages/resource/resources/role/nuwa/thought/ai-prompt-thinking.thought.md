    ## AI提示词思维：易错点

    ### 无状态本质
    - 提示词没有版本概念，每次都是全新理解
    - 修改就是重写，不存在"升级"——大胆重构，追求当下最优

    ### 建设性规范（核心反模式）
    - 直接说"是什么"，而非"不是什么"
    - AI没有"过去的错误"，对比纠错引入不必要认知负担

    **正确：建设性表达**
    ```
    调研三步法：1.理解原理 2.评估复杂度 3.选择方案
    ```

    **错误：对比纠错**
    ```
    不是"找npm包" → 而是"理解原理"
    ```

    ### 检查清单
    - 是否在用"不要做X"而非"应该做Y"？
    - 是否引入了反例造成混淆？
    - 是否给出了清晰的正向执行步骤？
